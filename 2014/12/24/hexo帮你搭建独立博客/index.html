hexo帮你搭建独立博客

毫无疑问,作为程序员搭建属于自己的独立博客是很有必要，记录自己的思想，反复斟酌，与他人交流想法，在教学相长下，我们都可以成长很多，回首再看，这或许是很值得的。

首先我为什么选择Hexo，而不是Octopress，原因在于Hexo，提供了方便的操作，Octopress还需要涉及到Git基本操作，所以为了简便性选择了Hexo，Hexo由台湾同胞编写，本人博客不是基本主题，大家可以在此基础上更换主题。

你需要这样几样东西

  1. GitHub 账号
  2. 将Hexo相关内容clone到本地
  3. 进行基本的Hexo更换主题or直接写博文

#提前准备

声明一点，所有操作都是Mac环境下

###安装Homebrew(在终端是输入，直接复制即可)

ruby -e "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/master/install)"

Homebrew官方文档

###安装Node.js

brew install node

若非最新版本，则可直接上官网直接下载.pkg文件下载地址

###安装Hexo

npm install -g hexo

一旦hexo安装好了，你首先要自己创建一个相关文件夹例如MyHexo，然后输入一下代码

hexo init ~MyHexo/
cd ~MyHexo/
npm install

#SSH配置
这三步完成后，需要解决与GitHub Page互相联系的问题
由于国内的特殊情况，上传博客有时候很不稳定，为此我可苦恼了很久，还以为是自己的操作的问题，╮(╯▽╰)╭，
若你电脑装了GitHub客户端，你就不用设置SSH了，若没有请按照一下步骤操作

###检查SSH keys的设置

cd ~/.ssh

###生成新的SSH Key

ssh-keygen -t rsa -C "邮件地址@youremail.com"
Generating public/private rsa key pair.
Enter file in which to save the key (/Users/your_user_directory/.ssh/id_rsa):<回车就好>

这里你要输入的是你在GitHub上注册的邮箱，然后根据提示输入加密串（这里随意）

Enter passphrase(empty for no passphrase):<输入加密串>
Enter same passphrase again:<再次输入加密串>

若成功了，会有相应的提示

###添加SSH

位置是 AccountSettings -> SSH Keys -> Add SSH Key

查看你之前生成的SSH文件，用文本编辑器打开ssh.pub，文件内容以ssh-rsa开头，复制进Key框内，点击Add Key即可

###测试添加SSH是否成功


ssh -T git@github.com

若有以下反馈就是成功了


The authenticity of host 'github.com (207.97.227.239)' can't be established.
RSA key fingerprint is 16:27:ac:a5:76:28:2d:36:63:1b:56:4d:eb:df:a6:48.
Are you sure you want to continue connecting (yes/no)?

输入yes即可

###设置个人信息

git config --global user.name "SoXeon"//用户名
git config --global user.email  "dpsopower@gmail.com"//填写自己的邮箱

#GitHub创建仓库
SSH设置完毕，就要在GitHub上面创建相应的代码仓库了
project name : yourname.github.io
description: 随意
ReadME可选
创建完毕基本就是本地工作啦

#hexo日常操作
为了早日看到博客的效果

你必须先进入你创建的那个目录下

hexo generate

这里会执行一系列创建操作，生成最基本的形态

hexo server

启动本地服务器你可以输入http://localhost:4000即可访问

##写博文

hexo new "pastname"

然后找到创建的源文件，markdown格式的文件，上网找相关的编辑器及入门教程

##更换主题
这里必须要在你创建的目录下

git clone https://github.com/heroicyang/hexo-theme-modernist.git themes/modernist

安装完毕后打开你创建的根目录下_config.yml文件

修改 theme:相应的主题名（modernist）

然后需要打开主题文件夹中的_config.yml文件，这里是 两个不同文件
编辑内容如下

menu: #配置页头显示哪些菜单
Home: /
Archives: /archives
Reading: /reading
About: /about
Guestbook: /about

excerpt_link: Read More #摘要链接文字
archive_yearly: false #按年存档

widgets: #配置页脚显示哪些小挂件
  - category
  - tag
  - tagcloud
  - recent_posts
  - blogroll

blogrolls: #友情链接


fancybox: true #是否开启fancybox效果

duoshuo_shortname:  #多说账号

google_analytics:
rss:

##更新主题

cd themes/modernist
git pull
